# 리구조화 플레이북

> Health Audit 결과에 따라 해당 패턴을 선택하고 실행한다. 모든 리구조화는 Plan 승인 후 수행.

---

## 패턴 1: SKILL.md 비대화 해소

### 증상
- SKILL.md가 500줄 초과
- 상세 규칙, 코드 예시가 SKILL.md 본문에 인라인

### 처방
1. SKILL.md에서 "판단 원칙" 추출 → `judgment/`
2. SKILL.md에서 "프로세스/절차" 추출 → `process/`
3. SKILL.md에서 "참조 데이터/패턴" 추출 → `references/`
4. SKILL.md에 동적 로딩 테이블 추가 (작업 유형 → 파일 매핑)
5. SKILL.md는 **역할 + ENFORCED 규칙 + Init Checklist + 동적 로딩 테이블**만 유지

### 분류 가이드

| 분류 | 질문 | 예시 |
|------|------|------|
| judgment/ | "어떤 판단을 내려야 하는가?" | 원칙, 의사결정 패턴, 리뷰 기준, 테스트 전략 |
| process/ | "어떤 순서로 실행해야 하는가?" | 요구사항 분석, PR 생성, 기능 딜리버리 |
| references/ | "어떤 데이터를 참고해야 하는가?" | 안티패턴, 코드 패턴, 레슨, 기술 참조 |

### 검증
- [ ] SKILL.md가 200줄 이하로 축소되었는가?
- [ ] 기존 내용이 누락 없이 참조 파일로 이전되었는가?
- [ ] 동적 로딩 테이블에서 모든 작업 유형이 커버되는가?

## 패턴 2: 참조 파일 분할

### 증상
- 단일 참조 파일 500줄 초과
- Index/TOC만으로는 부분 로딩이 불편

### 처방
1. 대상 파일의 섹션을 기능 단위로 분류
2. 각 섹션을 독립 파일로 분할 (100~300줄 목표)
3. 원본 파일 위치에 Index 파일 배치 (섹션 요약 + 링크)
4. SKILL.md의 동적 로딩 테이블에서 참조 경로 업데이트

### 검증
- [ ] 모든 분할 파일이 100~300줄 범위인가?
- [ ] Index 파일에서 모든 하위 파일에 접근 가능한가?
- [ ] nested reference가 1단계 이하인가?

## 패턴 3: 폴더 간 내용 재분류

### 증상
- judgment/ 파일에 프로세스성 내용 혼재
- process/ 파일에 참조 데이터 혼재
- 새로운 카테고리가 필요한 항목이 3개 이상

### 처방
1. 전 파일의 각 섹션에 태그 부여: `[judgment]`, `[process]`, `[reference]`
2. 태그 기준으로 재배치
3. 새 카테고리가 필요하면 폴더 신설 (단, 2단계 중첩 금지)

### 검증
- [ ] 각 폴더의 파일이 해당 분류의 성격에 맞는가?
- [ ] 새 폴더가 SKILL.md의 동적 로딩 테이블에 반영되었는가?

## 패턴 4: 레슨 졸업 배치

### 증상
- Active 레슨 10개 이상

### 처방
SKILL.md의 "레슨 졸업 배치 프로세스" 참조.

### 사후 검증

- [ ] 졸업한 레슨의 방지책이 해당 가이드에 실제로 추가되었는가?
- [ ] 졸업 아카이브에 기록이 추가되었는가?

## 패턴 5: 중복 가이드 병합

### 증상
- 2개 이상 파일에서 같은 주제를 다룸
- 파일 A에서 "상세는 파일 B 참조" + 파일 B에서 "상세는 파일 A 참조"

### 처방
1. 중복 파일 목록 작성
2. 주 소유 파일 결정 (더 상세한 쪽)
3. 부 소유 파일의 내용을 주 소유 파일로 병합
4. 부 소유 파일에서 중복 부분 제거 + 주 소유 파일 링크로 대체

### 검증
- [ ] 동일 주제가 단일 파일에 통합되었는가?
- [ ] 다른 파일에서의 참조 링크가 유효한가?

## 패턴 6: 신규 스킬 Bootstrap

### 최소 구조
```
[skill-name]/
└── SKILL.md    # 역할 + description + 핵심 절차 (50줄 이하로 시작)
```

### Growth 전환 체크리스트
- [ ] SKILL.md가 200줄 초과 → 하위 폴더 분리 시작
- [ ] 반복적 참조 데이터 3개 이상 → `references/` 분리
- [ ] 의사결정 원칙 5개 이상 → `judgment/` 분리
- [ ] 절차형 가이드 3개 이상 → `process/` 분리
- [ ] 레슨/실수 패턴 축적 시작 → `lesson-learnings.md` 생성

### Bootstrap 완료 기준
- [ ] description이 트리거 역할을 하고 있는가?
- [ ] body에 검증 루프가 포함되어 있는가?
- [ ] Claude가 이미 아는 내용을 장황하게 설명하고 있지 않은가?
