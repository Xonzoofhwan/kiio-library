# 건강성 점검 체크리스트

> 대상 스킬 디렉토리에 대해 4영역 점검을 수행하고, 결과를 지표 테이블에 기록한다.

---

## 사전 준비

```bash
# 대상 스킬의 파일별 줄 수 수집
wc -l $(find .claude/skills/[target-persona]/ -name "*.md" -type f) | sort -rn
```

---

## A. 구조 점검

### A1. SKILL.md 역할 검증
- [ ] SKILL.md가 500줄 이하인가?
- [ ] description이 트리거 역할만 하는가? (body에 "when to use" 없음)
- [ ] SKILL.md가 TOC 역할인가? (상세 내용은 참조 파일에)
- [ ] 동적 로딩 테이블이 있는가? (작업 유형 → 로드할 파일 매핑)

### A2. 파일 크기 분포
- [ ] 300줄 이상 파일 목록 작성
- [ ] 300줄 이상 파일에 Index/TOC가 있는가?
- [ ] 500줄 이상 파일은 분할 대상으로 표시
- [ ] 30줄 이하 파일 중 합치기 후보 식별

### A3. 참조 깊이
- [ ] 참조 파일에서 다른 참조 파일을 참조하는 경우 목록화
- [ ] 2단계 이상 nested reference가 있는가? → 평탄화 대상

### A4. 폴더 분류 정합성
- [ ] judgment/ 파일이 실제로 "판단 원칙"인가? (프로세스가 섞여 있지 않은가?)
- [ ] process/ 파일이 실제로 "절차"인가? (참조 데이터가 섞여 있지 않은가?)
- [ ] references/ 파일이 실제로 "참조 데이터"인가? (절차가 섞여 있지 않은가?)

## B. 내용 점검

### B1. 중복 검사
- [ ] 동일 주제가 2개 이상 파일에 분산되어 있는가?
- [ ] SKILL.md와 참조 파일 간 내용 중복이 있는가?

### B2. Claude-knows-already 검사
- [ ] Python/JS 기본 문법 설명이 장황하게 포함되어 있는가?
- [ ] 프레임워크 공식 문서의 재작성이 포함되어 있는가?
- [ ] → 코드 예시 + expected output으로 대체 가능한 부분 표시

### B3. MUST vs always 일관성
- [ ] 준수 필수 항목에 "MUST"가 사용되었는가?
- [ ] "항상", "always"가 "MUST"로 대체 가능한 곳 표시

## C. 레슨 시스템 점검

### C1. Active 레슨 현황
- [ ] Active 항목 수: ___개
- [ ] 10개 이상이면 졸업 배치 프로세스 실행 (SKILL.md 참조)

### C2. Embedded 레슨 유효성
- [ ] Embedded 레슨의 방지책 위치가 여전히 유효한가?
- [ ] 가이드 파일이 리구조화되면서 방지책이 사라진 곳은 없는가?

### C3. Enforced 레슨 작동 확인
- [ ] Enforced 항목의 자동 트리거가 실제로 작동하는가?

---

## D. 크로스 페르소나 일관성

### D1. 공유 패턴
- [ ] backend/frontend 양쪽에 적용되는 패턴이 한쪽에만 있는가?
- [ ] session-retrospective가 양쪽 스킬을 탐색 대상으로 포함하는가?

### D2. 성숙도 격차
- [ ] 한쪽 페르소나에만 레슨 시스템이 있는가? → 부트스트랩 권고
- [ ] 한쪽에서 졸업된 레슨이 다른 쪽에는 Active로 남아 있는가?

## 결과 기록

| 지표 | 값 | 상태 |
|------|------|------|
| SKILL.md 줄 수 | ___줄 | OK (≤200) / WARNING (201-500) / CRITICAL (>500) |
| 300줄 초과 파일 수 | ___개 (TOC 유: ___, 무: ___) | OK (모두 TOC) / WARNING |
| 500줄 초과 파일 수 | ___개 | OK (0) / CRITICAL (1+) |
| Active 레슨 수 | ___개 / 전체 ___개 (___%) | OK (≤40%) / WARNING (>40%) |
| nested reference 깊이 | 최대 ___단계 | OK (≤1) / CRITICAL (2+) |
| 중복 주제 수 | ___건 | OK (0) / WARNING (1+) |

### 판정

- **CRITICAL 1개 이상**: 즉시 리구조화 실행
- **WARNING 3개 이상**: 리구조화 계획 수립
- **모두 OK**: 다음 분기 점검까지 유지
