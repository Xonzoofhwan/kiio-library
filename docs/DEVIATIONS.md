# System Deviations Log

> 시스템(토큰·규칙·구조)에서 벗어난 판단을 기록한다.
> 동일 이탈이 3회 반복되면 시스템 수정을 검토한다. (원칙 8: 시스템 이탈 = 수정 신호)

---

## 기록 형식

### 필드 설명

- **날짜**: YYYY-MM-DD
- **컴포넌트**: 이탈이 발생한 컴포넌트 또는 토큰 (예: `Button`, `semantic.primary`, `spacing`)
- **이탈 내용**: 어떤 규칙에서 벗어났는지 구체적으로 기술
- **근거**: 이탈을 수용한 이유. 근거 없이 이탈한 경우 "근거 없음 — 수정 필요"로 기록
- **관련 원칙/규칙**: 위반한 원칙 번호 또는 규칙 문서 참조 (예: `F1 고정점 우선`, `B절 하드코딩 금지`)
- **반복 횟수**: 동일 패턴의 이탈이 누적된 횟수. **3회 도달 시 시스템 수정 검토를 트리거**한다
